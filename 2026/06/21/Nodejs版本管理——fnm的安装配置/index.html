Nodejs版本管理——fnm的安装配置

Created2026-06-21|Updated2026-06-21
|Word Count:1.9k|Reading Time:7mins|Post Views:|Comments:

前言

我之前的博客项目是少数依赖 Nodejs 的项目之一，平时主要使用 Hexo 写文章和部署博客。问题是，后面如果还想安装 Codex CLI、前端项目或者其他 Nodejs 工具，很容易遇到一个麻烦：不同项目需要不同版本的 Nodejs。

这就像 Python 项目会用 conda 管理环境一样，Nodejs 也需要一个版本管理工具。本文记录我在 Windows 下使用 fnm 管理多个 Nodejs 版本的完整过程：安装、配置、给博客项目锁定版本、继续使用 Hexo 指令，以及最后卸载旧的全局 Nodejs。

这意味着，只要系统 Nodejs 版本升级或删除，所有依赖 Nodejs 的项目都会被影响。对于博客项目来说，这不是很稳。

fnm 的作用就是管理多个 Nodejs 版本。每个项目可以使用自己的 .node-version 文件指定 Nodejs 版本，进入项目目录后自动切换。
