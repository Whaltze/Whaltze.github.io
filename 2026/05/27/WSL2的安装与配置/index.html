WSL2的安装与配置

🌟 Windows 用户的终极救赎：WSL2 零基础保姆级配置全指南（含进阶调优）

大家好！欢迎来到我的博客。

作为一个经常需要和各种控制系统、跨平台项目打交道的开发者，我深知装双系统时的提心吊胆，以及用传统虚拟机时的卡顿与笨重。直到微软推出了 WSL2 (Windows Subsystem for Linux 2)——它完美融合了 Windows 的漂亮 UI 和 Linux 强大的底层生态。

这篇指南专为完全没有 Linux 基础的新手准备，并融合了诸多进阶调优技巧。跟着这篇教程走，你能避开新手期 99% 的坑！🚀

💡 0. 什么是 WSL2 以及安装前置准备

在正式动手前，我们需要了解基础概念并检查电脑状态。

  • WSL 的全称是 Windows Subsystem for Linux，可以理解成让你在 Windows 里面直接使用 Linux。
  • 与初代相比，WSL 2 使用真正的 Linux 内核，对 Linux 软件兼容性更好，性能通常也更好。
  • 安装基本条件：你需要满足电脑系统是 Windows 11、拥有管理员权限，并且电脑开启了虚拟化。

如何确认电脑是否开启了虚拟化？

  • 按 Ctrl + Shift + Esc 打开任务管理器。
  • 点击“性能”选项卡，选择 CPU，在右下方查看“虚拟化”状态是否为“已启用”。
  • 如果没有开启：按下 Win+R，输入 optionalfeatures 打开 Windows 功能窗口。在弹出的窗口中，勾选“适用于 Linux 的 Windows 子系统”和“虚拟机平台”。等待安装完成后，务必点击“立即重启”，否则后续步骤会失败。

🛠️ 1. 安装与版本配置

安装 WSL2 只需要通过 Windows 的 PowerShell（以管理员身份运行）即可完成。

  • 更新 WSL 内核：首先更新 WSL 到最新版本，执行命令 wsl --update。如果下载很慢，可以使用 Web 下载命令 wsl --update --web-download。
  • 设置默认版本：确保默认使用的是第二代内核，执行 wsl --set-default-version 2。
  • 查看与安装：
  • 显示当前支持在线安装的 Linux 发行版列表：wsl --list --online。
  • 安装指定的系统（以 Ubuntu 22.04 为例）：wsl --install -d Ubuntu-22.04。

等待解压完成后，终端会提示你创建用户（全小写英文字母）和密码。注意：输入密码时屏幕上不会显示任何字符，盲打完按回车即可。

💽 2. C盘拯救计划：迁移发行版到指定目录

默认情况下，系统会安装在 C 盘上，但你可以将其迁移到其他位置以节省空间。

  1. 关闭所有实例：关闭所有正在运行的 WSL 实例，避免迁移时文件占用。
  1. 导出为压缩包：把名为 Ubuntu-22.04 的发行版导出为一个 tar 备份文件，保存到指定位置（例如 E 盘）。
  1. 注销原系统：注销系统盘里的旧发行版。
  1. 重新导入到新位置：把导出的备份导入到指定的新目录，并指定使用 WSL 2。

⚙️ 3. 系统基础配置与 Systemd 启用

进入 Ubuntu 后，我们需要做一些基础的安全和启动项配置。

* 安装并自启 SSH：在 WSL 中运行 sudo apt install openssh-server，然后通过 systemctl enable ssh 和 systemctl start ssh 让 SSH 开机自启。

🌐 5. 网络与软件源配置

  • 配置网络环境：如果你的 Windows 11 版本较新，可以在刚才的 .wslconfig 中加入 networkingMode=mirrored 和 autoProxy=true，让 WSL 完美继承 Windows 的网络和代理，解决下载慢的问题。

  • 更换国内软件源：Ubuntu 默认源是国外服务器，国内访问速度慢，替换为阿里源或清华源，更新和下载软件会快很多。
    • 使用 sudo nano /etc/apt/sources.list 清空默认源并粘贴阿里源。
