安装配置Miniforge

第一步：下载正确的安装包

直接从 GitHub 官方仓库下载最新的 Windows x86_64 版本。

  1. 访问地址： Miniforge GitHub Releases & Forge-org
  2. 目标文件： 找到列表中的 Miniforge3-Windows-x86_64.exe (通常在 Latest Release 下)。
  3. 下载： 下载到本地。

注意 Miniforge3-25.11.0-1-Windows-x86_64.exe.sha256 文件是哈希校验文件, 用于与安装包文件进行比较, 检验安装包是否完整

第二步：安装过程（关键避坑点）

双击运行安装包，一路 Next，直到遇到以下两个关键界面，请务必按我说的做：

  1. Select Installation Type (安装类型):
  • 选择 **Just Me (recommended)**。
  • 理由： 避免权限问题，将文件限制在你的用户目录下，不影响系统其他用户。
  1. Advanced Options (高级选项) —— 这里最重要！

  • 🔲 Create shortcuts… (创建快捷方式): ✅ 勾选。

  • 🔲 Add Miniforge3 to my PATH environment variable (添加到 PATH): ❌ 绝对不要勾选。

  • 这是保持系统干净的核心。如果不勾选，CMD 和 PowerShell 默认找不到 conda，只有通过专用的快捷方式才能启动，实现了物理隔离。

  • 🔲 Register Miniforge3 as my default Python 3.12… (注册为默认 Python): ❓ 可选，建议不勾选。

  • 为了极致的隔离，建议不勾选。如果你后续安装 VS Code，我们在 VS Code 内部指定路径即可，不需要通过注册表告诉全世界它的存在。

  1. 安装路径： 默认通常是 C:\Users\你的用户名\miniforge3。建议保持默认，或者改到空间较大的盘（如 D:\Dev\miniforge3），但路径千万不要包含中文或空格。

第三步：验证安装与初始化

安装完成后，你的“开始菜单”里会出现一个名为 Miniforge3 的文件夹。

  1. 启动专用终端：
  • 点击 Miniforge Prompt。
  • 你会看到一个黑色的命令行窗口，最左侧显示 (base)。这代表你现在处于 Miniforge 的基础环境中。
  1. 验证隔离性：

  • 在 Miniforge Prompt 里输入：where python

  • 结果应指向： ...\miniforge3\python.exe

  • 打开一个普通的 PowerShell 或 CMD，输入：python

  • 结果应该是： 找不到命令，或者跳出 Windows 应用商店。

  • 结论： 恭喜，你的系统环境依然是一张白纸，Miniforge 被完美关进了笼子。

第四步：配置“生产级”开发习惯

在 Miniforge Prompt 中执行以下命令，优化你的使用体验。

1. 禁止 Base 环境自动激活（防止误操作）

Base 环境只应该用来管理 conda 本身（升级 conda、创建新环境），永远不要在 Base 里安装你的开发包（numpy, pandas 等）。

效果：下次打开终端，如果你不主动 conda activate，就不会进入任何环境，避免你迷迷糊糊把包状错位置。

2. 配置下载加速（如果你在国内）

由于 Miniforge 默认连接的是国外的 conda-forge，下载速度可能很慢。我们可以配置清华大学或北京外国语大学的镜像源。

第六步：连接 IDE (VS Code)

作为开发者，你肯定不会一直用黑框框写代码。推荐安装 VS Code (通过官网下载安装即可，这个不污染环境)。

如何让 VS Code 识别你的 Miniforge 环境？

  1. 用 VS Code 打开你的代码文件夹。
  2. 打开一个 .py 文件。
  3. 点击右下角的 Python 版本号（或者按 Ctrl+Shift+P 输入 Python: Select Interpreter）。
  4. VS Code 通常会自动扫描到 ...\miniforge3\envs\algo_dev\python.exe。
  5. 如果没有扫描到，点击 “Enter interpreter path”，手动找到你 Miniforge 安装目录下的 envs 文件夹即可。

总结

现在，你已经拥有了一个极其干净、专业、可移植的 Windows Python 开发底座：

  • 系统 PATH： 干净（无 Python）。
  • 注册表： 干净。
  • 依赖管理： 全部在 miniforge3 文件夹内闭环。
  • 未来扩展： 需要 C++ 编译器？conda create -n cpp_env 然后 mamba install compilers 即可，互不干扰。

第二步：修改 PowerShell 脚本执行权限（必需）

即使初始化成功，若 Windows 禁止加载配置文件，下次打开终端仍会报错。
在当前终端中继续输入：

命令详解：Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

可以将该命令拆解为三部分理解：

  1. **Set-ExecutionPolicy**（设置执行策略）

    • 含义：修改 PowerShell 的安全规则。
    • 背景：Windows 默认策略为 Restricted（受限），即禁止运行任何脚本文件。
    • 后果：Conda 启动时需要运行脚本（如 profile.ps1），因此被系统拦截。

  2. **-ExecutionPolicy RemoteSigned**（策略设为：远程签名）

    • 含义：新规则的具体内容。
    • RemoteSigned 规则：
      • 本地创建的脚本（如自己编写或 Conda 生成的）：允许直接运行，无需签名。
      • 从互联网下载的脚本：必须有受信任发布者的数字签名方可运行。
    • 作用：在保障开发便利（允许使用 Conda、Python 等工具）的同时，防止自动运行恶意脚本。此为开发者常用设置。

  3. **-Scope CurrentUser**（范围：仅当前用户）

    • 含义：该规则仅对当前登录的 Windows 用户生效。
    • 优点：
      • 安全性高：不影响其他用户。
      • 通常无需管理员权限：可直接修改（若电脑管控严格，可能仍需提权）。
